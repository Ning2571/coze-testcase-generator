# 基于Coze的测试用例自动生成工作流

## 这是什么？

一个基于 **Coze（扣子）** 零代码平台搭建的测试用例自动生成工具。  

**核心功能**：上传需求文档（Word / PDF），自动输出一份**带优先级划分**（P0 / P1 / P2）和**测试维度标注**（正向 / 边界 / 异常）的结构化测试用例思维导图，帮助测试人员快速搭好用例框架，把精力留给更复杂的逻辑校验和边界挖掘。

## 项目背景

在测试工作中，从需求文档整理出测试用例大纲是一件重复且耗时的事情，初学者也很容易漏掉某个测试维度。  
为了把这件事自动化，我在 Coze 上用 **4 个节点** 搭建了这条工作流，半小时就完成了核心串联，又花了约八小时反复调试系统提示词，让生成的用例更贴合真实测试标准。

## 怎么用？

1. 打开 Coze 工作流链接（见下方）  
2. 上传 Word 或 PDF 格式的需求文档  
3. 等待大模型解析并生成测试用例  
4. 下载或查看生成的思维导图，导入飞书多维表格进行用例管理和缺陷跟踪

### 4. 数据清洗与格式化

生成的用例导入飞书多维表格后，部分行的“测试维度”列出现了格式问题：优先级标记和维度描述混在一起（如 `P1【异常校验】`），无法直接用于筛选和管理。

我没有逐行手动修改，而是直接调用飞书内置AI助理，给出了一条精确的指令：

> “将表格中32至85行、测试维度列里所有包含‘P1【异常校验】’的单元格，删除其中的‘【’、‘】’符号和‘P1’字符，仅保留‘异常校验’。”

几秒内完成全部清洗，表格恢复为标准格式。


## 项目价值

- **痛点解决**：将“从零写用例大纲”的重复劳动自动化，AI 负责搭框架，人负责审核和补充
- **质量约束**：通过精心调试的系统提示词，让 AI 按“正向/边界/异常”维度和 P0/P1/P2 优先级产出用例
- **工程化落地**：生成的用例不是终点，而是被导入飞书多维表格，变成可管理、可跟踪的测试资产
- **最后防线**：每一条用例都必须经过人工审核，这是基于我两次实习审核经验定下的底线
